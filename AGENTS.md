# Project: MadRank — SEO & AI-search audit

A per-URL SEO + AI-search auditor with a Semrush-style dashboard. **JavaScript, not TypeScript.** Next.js 16 (App Router, React 19, React Compiler, Turbopack) + Tailwind v4 + shadcn/ui. Deployed on Vercel (team scope `madarth`, project `meta-tag`, prod https://superengine.vercel.app).

## How it works

`src/lib/seo/analyze.js#runScan(url, {deepScan})` is the orchestrator. It runs **synchronously per request** (no background jobs yet, capped at `MAX_PAGES_SYNC = 40`) and returns one `result` object that every UI panel + export consumes unchanged:

```
fetchSitemap + fetchAiSiteContext → analyze each page (fetch → extract → score)
→ optional headless escalation for JS-rendered pages → aggregate
→ { rootUrl, sitemap, pages[{url, signals, audit, aiAudit, ...}], aiReadiness, analytics, headless, ... }
```

A scan is run from **`/api/seo/audit`** (the `/seo` dashboard's audit bar; also reused for the cron monitor). It calls `saveScan(result, userId)` (best-effort) to persist to Neon and get a share token. **There is no standalone `/scan` route anymore** — it was deleted; its dashboard + panels (`src/app/scan/*`) live on purely as reusable components consumed by `/seo` and the shared-report page `/r/[token]`.

Library modules (`src/lib/seo/`): `safe-fetch` (SSRF-guarded fetch — use for ALL outbound requests; literal-host checks **and** DNS resolve-and-validate), `sitemap`, `crawl`, `headless` (puppeteer-core / Browserbase), `extract` (cheerio → signals), `rules` (classic SEO scoring), `ai-rules` (AEO/GEO/AIO/AGO), `ai-site` (llms.txt/ai.txt validation + AI-bot policy), `trackers`, `explanations` (plain-language per ruleKey), `gamify` (issue grouping), plus `readability`, `relevance`, `links-probe`, `keywords`, `backlinks`/`serp`/`dataforseo`/`competitors`, `markdown-report`, `google-updates`. Persistence: `src/lib/db.js` (Neon client, null when `DATABASE_URL` unset) + `src/lib/db/{scans,profiles,content,ai-settings,monitors,records}.js`. Rate limiting: `src/lib/rate-limit.js` (Neon fixed-window, best-effort). AI calls go through the Vercel AI Gateway or a user's BYO key (`src/lib/ai/*`, `api/ai-fix`).

UI: the reusable dashboard panels live in `src/app/scan/` (`scan-dashboard.js` + `app-sidebar.js` + one `*-panel.js` per section + `score-ring`/`og-preview`/`snapshot-cards`/`ai-fix`). The **live audit surface is `/seo`** (see below). Saved/shared reports at `src/app/r/[token]/` (renders `ScanDashboard`), scan history at `src/app/history/`. The marketing landing (`src/app/page.js`, "scan readout / instrument" design — Space Grotesk display, command-bar audit form, the AI Visibility Spectrum signature) and the global app shell (`AppShell` + `AppNavSidebar` + `SiteSwitcher`) wrap the app. Product analytics via `src/components/posthog-provider.js` (mounted in `layout.js`).

**Active-site context** (`src/lib/site/active.js` + `src/components/active-site-provider.js` + `src/app/dashboard/`): a "brand profile" (see Content studio) is the user's **active site** — the most-recently-used one (`brand_profiles.last_used_at`). `getActiveSite()` resolves it in the root `layout.js` and threads it (via `ActiveSiteProvider`) into the sidebar **site switcher**, the content tools' default profile, and the scan entry's prefilled URL. Running an audit stores the audited site in this store (touch-or-create). `/dashboard` is the okara-style home: it reuses the active site's latest cached scan to surface top fixes + AI-readiness and routes into each "agent". `/account` shows identity + sign-out + Google connect/disconnect.

**Auth + access** (`src/lib/auth/` + `src/app/api/auth/*` + `src/app/login/`): Google sign-in (raw OAuth, mirrors the GSC pattern; scopes `openid email profile` — non-sensitive, no Google verification; OAuth `state` validated, `next` constrained to relative paths). `session.js` stores `users`/`sessions` in Neon (24-byte token, 30-day expiry), `app_session` httpOnly+sameSite=lax cookie, `currentUser()` server helper. **Plans were removed — `plan.js#planOf()` returns full access for everyone** (deepScan + 40 pages); there is no Free/Pro tier and no billing. Gating is server-side via `currentUser()` (no middleware): scanning + the content tools require login. The whole auth feature **no-ops when `GOOGLE_CLIENT_ID`/`SECRET` are unset** (everything stays open). Same Google client as GSC — register **both** `…/api/auth/callback` and `…/api/gsc/callback` redirect URIs.

**`/seo` — the unified audit + Analytics/Search Console dashboard** (`src/app/seo/`): this is now the app's **single audit surface** (the old `/scan` was folded in here). One **unified per-visitor Google OAuth** connection powers the live data; `GSC_SCOPE` in `oauth.js` requests `webmasters.readonly` **+ `analytics.readonly`** + `openid email`, stored in the same `gsc_tokens` row / `gsc_session` cookie (already-connected users reconnect once; `prompt:"consent"` handles it). `/search-console` 307-redirects here; nav label is "SEO".
- **7-tab dashboard** (`seo-dashboard.js` + `*-tab.js`): **Traffic** (live GA4 + GSC via `GaDashboard`/`GscDashboard` + a combined funnel from `/api/seo/funnel`), and **SEO / Pages / Links / Technical / GEO / Tracking** — the audit tabs, which **reuse the scan panels** (`@/app/scan/{overview,pages,issues,ai-readiness,performance,tracking}-panel`, `score-ring`, `og-preview`, `snapshot-cards`) against one shared audit result via `ScanContext` (`scan-context.js` + `scan-gate.js`).
- **Audit bar (URL entry, Google-optional)**: type **any** URL (prefilled from the active site) + a deep-scan toggle → `/api/seo/audit?url=…&full=1&deep=1` runs a full (optionally deep, multi-page) `runScan`, **no Google connection required** (login-gated only). Opening an audit tab also lazily runs a cheap single-page auto-audit of the active site. `/api/seo/audit` also still accepts `?site=` (a connected GSC property, via `siteUrlForScan`, requires `gsc_session`). It reuses a ≤60-min saved scan (`latestScanForUrl`) for cheap views, forces fresh on explicit/deep runs, returns a share token, and tags the active-site store on explicit runs. The Traffic tab is the only part that needs Google — it prompts to connect when not linked. The page renders even without a Google connection (or without GSC configured at all).
- **Net-new audit signals** feeding the Technical/SEO/Links tabs: `safeFetch` now returns `server`/`contentEncoding`/`cacheControl`/`cacheable`/`byteSize`/`timings`; `extractSignals` adds `headings.byLevel`, `renderBlocking`, `contentRatio`, `domNodes`(+`domNodesApprox`), `htmlIssues`, `readability` (`src/lib/seo/readability.js` — Flesch, returns null on non-prose), `relevance` (`src/lib/seo/relevance.js` — token overlap), and `linkUrls`; `runScan` probes the homepage's links (`links-probe.js` → `signals.linkSample`) and rolls up `result.contentSummary`. All pure, all null-guarded, no DB migration.
- **Search Console** (`src/lib/gsc/` + `src/app/api/gsc/*`): `oauth.js` (auth URL / code exchange / refresh / id_token email), `tokens.js` (Neon `gsc_tokens`; refresh token AES-256-GCM-encrypted with a key derived from `GOOGLE_CLIENT_SECRET`; `getValidAccessToken` auto-refreshes — **reused by GA**), `api.js` (Search Analytics REST + pure transforms: totals, deltas, trend, top queries/pages, striking-distance). `isGscConfigured()` (needs `GOOGLE_CLIENT_ID/SECRET`) gates both halves.
- **Google Analytics (GA4)** (`src/lib/ga/` + `src/app/api/ga/*`): `api.js` — Admin API `accountSummaries` (`listProperties`) + Data API `runReport` (`buildGaReport`: totals/deltas/trend + top pages/channels/countries, conservative metric set so runReport never 400s). Routes read the **same** `gsc_session` via `getValidAccessToken`. Reports persisted best-effort to `ga_reports` (`saveGaReport`). **Enable the Analytics Data API + Analytics Admin API** in Google Cloud (alongside the Search Console API).

**Site health snapshot — backlinks + live SERP** (`src/lib/seo/backlinks.js` + `serp.js` + `keywords.js` + `src/app/api/seo/{backlinks,serp}/*` + `src/app/scan/snapshot-cards.js`): the scan **Overview panel** leads with a consolidated snapshot — SEO score + Lighthouse performance (live PageSpeed) + **backlinks** + **live SERP rank**. Backlinks are **wired to DataForSEO** (`backlinks.js#fetchDataForSeo` — POST `/v3/backlinks/summary/live` + `/referring_domains/live`, Basic auth; direct `fetch` since the host is constant/trusted and `safeFetch` is GET-only); set `DATAFORSEO_LOGIN`/`DATAFORSEO_PASSWORD` to activate (else `{configured:false}` → "connect a provider" card). **SERP** is still provider-stubbed (same DataForSEO creds will drive it once `fetchDataForSeo` in `serp.js` is fleshed out). Because provider calls are **paid**, the snapshot only does a cheap `?check=1` capability probe on render and **button-gates** the real fetch (`Load backlinks` / `Load live rank`); routes reuse a ≤24h snapshot (`backlinks_snapshots`/`serp_snapshots`). SERP keywords come from `candidateKeywords` (on-page, no GSC). Note: GSC `position` is *self-reported* avg rank; this is *live* Google rank.

**Competitor discovery** (`src/lib/seo/competitors.js` + `src/lib/seo/dataforseo.js` + `src/app/api/seo/competitors/route.js` + `src/app/competitors/*`): the `/competitors` page auto-discovers competitors for a domain two ways and merges them — **DataForSEO Labs `competitors_domain`** (SERP-overlap ranked, `intersections` → common keywords) + an **AI-gateway LLM-ask** (`generateText` structured output; brand names/breadth). Either half is optional (`isCompetitorsConfigured()` = `DATAFORSEO_*` **or** `AI_GATEWAY_API_KEY`); no-ops to `{configured:false}`. The DataForSEO `dfsPost`/auth client was extracted from `backlinks.js` into the shared **`dataforseo.js`** (`backlinks.js` re-exports `domainOf`). Discovery is **paid** → button-gated (`?check=1` probe) + ≤24h `competitor_snapshots` reuse. The domain input prefills from `/api/gsc/sites`; selected competitors (max 3) deep-link into `/compare?url=target&url=…`. Login-gated like `/compare`.

**Link intelligence** (`src/lib/seo/{extract,links-probe,analyze,rules,crawl}.js` + `src/app/seo/links-tab.js` + `src/app/scan/overview-panel.js`): beyond counts, the scan now computes **outbound link quality** (`signals.outboundQuality` {dofollow,nofollow,sponsored,ugc,emptyAnchor} + `externalSample` with rel/anchor), **site-wide broken-link + redirect health** (`result.linkHealth` — `links-probe` follows redirects manually recording `redirectChain`/`finalUrl`; aggregated across all analyzed pages, cap 40; a `links.broken`/`links.redirect_chain` rule in `rules.js` feeds the Issues panel, and the root page is **re-scored** after probing so it counts toward `siteScore`), and an **internal-link graph** on deep scans (`result.internalGraph` — `crawl.js` now returns `graph{depthByUrl,inboundCounts}`; `analyze.js#buildInternalGraph` derives orphans, click-depth, top-linked). Surfaced in the /seo **Links tab** (outbound-quality + redirect-aware broken table) and the scan **Overview** (internal-link-structure card; deep-scan only).

**AI content studio — brand memory + articles + social posts** (`src/lib/ai/{generate-article,generate-social,models,crypto,user-model,errors}.js` + `src/lib/db/{profiles,content,ai-settings}.js` + `src/app/{profiles,articles,post-ideas,ai-settings}/*` + `src/app/api/{profiles,articles,post-ideas,ai-settings}/*`): a per-user content generator driven by a reusable Markdown "memory". **Brand Memory** (`/profiles`) stores multiple named Markdown profiles (`brand_profiles` table) describing a site's voice/audience/topics; that Markdown is injected as the brand-context block into every generation. **Articles** (`/articles`) writes a full SEO/GEO article (structured `{title,slug,metaDescription,sections[],tags[]}` via `generateText` + `Output.object` + zod, rendered natively + assembled to Markdown for copy/download). **Post Ideas** (`/post-ideas`) generates platform-native social posts (Instagram/X/LinkedIn/Facebook/TikTok/Threads). Both **save to history** (`generated_content` table, `kind` = `article|social`, `status` = `draft|approved` — a review-and-approve loop with an Approve button + `PATCH /api/content/[id]`). **Login-only** (no plan tiers) — routes use `currentUser()` and fall back to anonymous `user_id NULL` rows when auth is unconfigured (like scanning). **Bring-your-own AI** (`/ai-settings`): users pick a provider (OpenAI/Anthropic/Google via `@ai-sdk/*`) + model + paste that provider's key; the key is **AES-256-GCM encrypted at rest** (`crypto.js`, key derived from `AI_KEY_SECRET` → falls back to `GOOGLE_CLIENT_SECRET`) in `ai_settings`, **never returned to the client** (GET exposes only `hasKey`). `user-model.js#userModel(userId)` resolves the stored provider/key into an AI SDK model instance; generators accept a `model` param and fall back to `DEFAULT_MODEL` (gateway string `anthropic/claude-haiku-4.5`, needs `AI_GATEWAY_API_KEY`/OIDC) when none is set. Model registry + curated model lists (with custom-id override) live in `models.js`; friendly auth/quota error mapping in `errors.js`. No-op-clean without DB/AI key.

## Conventions

- **JavaScript** (`.js`/`.jsx`), no TS. Path alias `@/*` → `src/*` (jsconfig). Match the surrounding code style.
- Next.js 16: `searchParams`/`params` are **Promises — await them**. API = `route.js` Route Handlers. Server Components do data fetching; mutations via Server Actions / route handlers.
- All new SEO logic stays as **pure functions** in `src/lib/seo/` operating on the `result`/`signals` shape (testable, reused by UI + exports). Don't recompute in components.
- Outbound HTTP must use `safeFetch`/`assertSafeUrl` (SSRF protection — users submit arbitrary URLs).
- Theme/severity colors come from CSS tokens in `globals.css` (`--primary` is the orange accent; `--pass/--warning/--error/--info`). Avoid ad-hoc hex.

## Gotchas (these have bitten us)

- **shadcn here is Base UI (`@base-ui/react/*`), NOT Radix.** `Accordion` uses `openMultiple` (default `true`) + `defaultValue` array — **never** `type="multiple"`. Button/Badge use Base UI `render`/`useRender`.
- **Turbopack dev serves stale CSS** after external edits (e.g. `sed`). After changing tokens/CSS, `rm -rf .next` and restart, then verify computed styles.
- **Tailwind v4 Preflight** already resets margins/box-sizing — no manual `* {}` reset. Buttons get `cursor: pointer` from an `@layer base` rule (Preflight v4 dropped it).
- The **`DEP0205` deprecation** warning comes from `@tailwindcss/node` and is suppressed via `NODE_OPTIONS=--disable-warning=DEP0205` in the npm scripts — keep that.
- `puppeteer-core` is in `next.config.mjs` `serverExternalPackages`. Headless runs when `BROWSER_WS_ENDPOINT`, `CHROME_EXECUTABLE_PATH`, or `BROWSERBASE_API_KEY`+`BROWSERBASE_PROJECT_ID` is set (Browserbase creates a session → `puppeteer.connect`); otherwise the scan stays fetch-only and flags `needsHeadless`. Escalation cap `MAX_HEADLESS = 5`.
- **Keyless PageSpeed Insights 429s** (shared quota) — Performance needs `PAGESPEED_API_KEY`. The key must **not** be HTTP-referrer-restricted (server-side calls send no Referer → Google 403s "Requests from referer <empty> are blocked"). `api/pagespeed` sends a `Referer` header (`PAGESPEED_REFERER` env, else the app origin) so a domain-restricted key works; otherwise set the key's restriction to None/IP in Google Cloud.
- AI features work locally with `AI_GATEWAY_API_KEY` and in prod via Vercel OIDC. Always fetch current gateway model IDs (`curl https://ai-gateway.vercel.sh/v1/models`) — don't hardcode from memory.
- **Persistence is best-effort**: all `src/lib/db/*` calls no-op (return null/`[]`) when `DATABASE_URL` is unset, so scans still work without a DB. Base schema lives in `db/schema.sql` (gitignored) → `node --env-file=.env.local scripts/migrate.mjs`; additive feature migrations are standalone idempotent scripts: `scripts/migrate-active-site.mjs` (`brand_profiles.last_used_at` + `generated_content.status`) and `scripts/migrate-rate-limits.mjs` (`rate_limits` table). Runtime only needs `DATABASE_URL`.
- **Security hardening** (OWASP pass): outbound fetch is SSRF-guarded by literal-host checks **and** DNS resolve-and-validate (blocks IPv6-mapped-IPv4 like `[::ffff:127.0.0.1]` and public-name→private-IP rebinding) in `safe-fetch.js`. Expensive routes (`seo/audit`, `ai-fix`, `articles`, `post-ideas`, `pagespeed`, paid `seo/{backlinks,serp,competitors}`) are rate-limited via `rate-limit.js` (per-user or per-IP, returns 429). `next.config.mjs` sets security headers (CSP with `frame-ancestors 'none'`, HSTS, nosniff, X-Frame-Options, Referrer-Policy, Permissions-Policy) — header/config changes need a dev-server restart. The cron secret compare is `timingSafeEqual`.
- **PostHog env vars are oddly lowercased**: `NEXT_PUBLIC_superengine_POSTHOG_PROJECT_TOKEN` / `_HOST` (the `superengine` segment is literal — Next inlines `NEXT_PUBLIC_*`). The provider no-ops cleanly without a token. Key events: `scan_run`, `share_created`, `ai_fix_used`, `performance_test_run`.
- **Vercel env gotcha**: the Neon Marketplace integration's `POSTGRES_*` vars were NOT auto-wired to the project — `DATABASE_URL` + `BROWSERBASE_*` + `NEXT_PUBLIC_superengine_POSTHOG_*` were added manually (production + development). Preview env is unset (CLI 54.9.1 ignores `--yes` on the all-branches path; upgrade to 54.12.1+).
- **`scripts/migrate.mjs` splits `db/schema.sql` on naive `;`** — never put a semicolon inside a SQL comment there or the splitter breaks the statement. Schema is idempotent (`CREATE TABLE IF NOT EXISTS`); re-running is safe.
- **Search Console is a Google *sensitive* scope** (`webmasters.readonly`): until the OAuth app passes Google verification, only console-added **test users** can connect (they see an "unverified app" screen). Feature no-ops without `GOOGLE_CLIENT_ID`/`_SECRET` + Neon. Register redirect URI `…/api/gsc/callback` (localhost + prod) in the Google console.
- **BYOK AI keys are encrypted with `AI_KEY_SECRET`** (falls back to `GOOGLE_CLIENT_SECRET`). Set a dedicated `AI_KEY_SECRET` in prod — **if it ever changes, previously stored provider keys can't be decrypted** (users just re-enter them, no crash). The plaintext key is never returned by the API. User-selected models can be arbitrary strings (custom-id input), so a bad/stale model id surfaces as a provider error via `errors.js`, not a crash.
